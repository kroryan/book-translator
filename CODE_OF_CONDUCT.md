# Code of Conduct

## Our Standard

Participation in this project should stay respectful, constructive, and professional.

Examples of expected behavior:

- be respectful in discussions and reviews
- focus on the code and the problem, not the person
- accept feedback without hostility
- give clear technical reasoning when disagreeing
- help keep the repository welcoming to new contributors

Examples of unacceptable behavior:

- harassment or personal attacks
- insults, intimidation, or discrimination
- bad-faith arguments or repeated hostility
- publishing private information without permission
- disruptive behavior that makes collaboration harder

## Enforcement

Project maintainers may remove comments, close threads, reject contributions, or block users whose behavior harms the project or its contributors.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainer through the repository contact details or open a private security-style report if public discussion would make the situation worse.

## Scope

This Code of Conduct applies to GitHub issues, pull requests, discussions, reviews, and other project communication spaces.
